Product Vision & Feature Blueprint

Executive Summary
MemoCare is a context-aware, chain-triggered smart reminder app designed primarily for elderly users, but adaptable for young professionals, students, and children. Unlike generic alarm apps, MemoCare understands the relationship between tasks — so reminding someone to take a pill is linked to whether they have eaten, and an Iftar reminder cascades into a medicine reminder chain.
The core innovation is the concept of Linked Reminder Chains: once the user confirms one step is done, the next in the sequence is automatically scheduled. This creates a guided daily care flow rather than a pile of disconnected alarms.

2. Core Concept: Linked Reminder Chains
A Linked Reminder Chain is a group of reminders where the firing of Reminder N depends on the user confirming Reminder N-1 is complete. The user sets one 'anchor event' (e.g., meal time) and the app calculates all satellite reminders automatically.

EXAMPLE
User says: 'I eat lunch at 1:00 PM. I need to take Paracetamol 500mg 5 minutes before, and Dolo 650 exactly 30 minutes after.' — App schedules: 12:55 PM alert (Paracetamol). User taps DONE. App then locks in 1:30 PM alert (Dolo). If user never taps DONE, app re-alerts at 1:00 PM and again at 1:05 PM with escalation.

2.1 Chain Trigger Logic
Feature
Description
Anchor Event
A fixed or estimated time (meal, prayer, wakeup, sleep, school bell)
Pre-Trigger
Fires X minutes BEFORE anchor (e.g., take tablet 5 min before eating)
Post-Trigger
Fires X minutes AFTER anchor confirmation (e.g., take syrup 30 min after meal)
Conditional Trigger
Fires ONLY if user confirmed a prior step (e.g., only remind about Dolo if they confirmed they ate)
Escalation Trigger
Fires again after a snooze window if no confirmation received
Skip Trigger
If user marks 'skipped meal', all meal-linked medicine reminders are suspended for that slot

2.2 Confirmation States
	•	DONE — Confirmed complete, next in chain fires as scheduled
	•	SNOOZED — Re-fire in 5 / 10 / 15 min (user picks snooze window)
	•	SKIPPED — Mark this reminder done without triggering the downstream chain
	•	MISSED — No response detected within window; caregiver notification sent

3. Reminder Types & Templates
3.1 Medical Reminders
Medical reminders are the most critical category and support the most complex chain logic.
Feature
Description
Before Meal Medicine
Fire X min before anchor meal. Requires explicit confirmation before downstream fires.
After Meal Medicine
Fire X min after anchor meal confirmation. Works even if meal time drifts.
Empty Stomach Medicine
Fire Y min after wakeup, only if no meal logged in prior 8 hours.
Fixed Time Medicine
Fires at a hard time regardless of meals (e.g., blood pressure pill at 8 AM always).
Dose Gap Medicine
Fires only if minimum gap since last confirmation has elapsed (e.g., Paracetamol every 6h).
Course Tracker
Multi-day medicine with progress bar (e.g., Day 3 of 7 antibiotic course).

3.2 Meal & Hydration Reminders
	•	Fixed meal reminders: Breakfast / Lunch / Dinner at user-set times
	•	Hydration reminders: Customisable interval (e.g., every 45 min) with daily count target (e.g., 8 glasses)
	•	Snack reminders for diabetics or children at regular intervals
	•	Appetite prompt for elderly users who skip meals unintentionally

Feature

4. Advanced Yet Easy Setup Mechanism
The biggest challenge is making a powerful scheduling system accessible to elderly users. MemoCare uses three input methods that all produce the same structured reminder behind the scenes.
4.1 Conversational Setup (Voice / Text)
User speaks or types in plain language. The app's NLP layer parses intent and builds the chain automatically.

USER SAYS
"I take Paracetamol 500mg five minutes before lunch and Dolo 650 half an hour after lunch. Lunch is around 1 PM."

APP CREATES
Anchor: Lunch ~ 1:00 PM | Chain 1: 12:55 PM — Paracetamol 500mg (pre-meal) | Chain 2: 1:30 PM after DONE confirmation — Dolo 650 (post-meal)

4.2 Template-Based Quick Setup
Pre-built templates for common patterns that users can activate in one tap and then fill in medicine names / times:

Description

Blood Pressure Pack
Morning BP pill on empty stomach + evening pill + BP measurement twice daily
Ramadan Medicine Pack
All medicines redistributed to Sehri and Iftar chains automatically

4.3 Visual Timeline Builder
For more technical users or caregivers setting up the app on behalf of an elderly person. A drag-and-drop day timeline where reminder blocks can be placed and linked with arrows showing chain dependencies. Chain links can be adjusted, medicine names typed in, and a 'test run' simulates the day's notification flow.
4.4 Smart Pattern Detection
	•	If user confirms 'Lunch done' at 1:15 PM three days in a row, app suggests shifting the anchor from 1:00 to 1:15 PM
	•	If medicine is consistently snoozed by 10 minutes, app offers to adjust the trigger time
	•	If user skips Breakfast often on Mondays, app asks if Monday schedule should differ

5. Notification & Alert Design
5.1 Alert Layers (Escalation Stack)
Feature
Description
Layer 1 — Soft Alert
Silent notification banner with medicine name, dose, and context ('5 min before your lunch')
Layer 2 — Audio Alert
Sound + vibration after 2 minutes of no response
Layer 3 — Loud Alert
Full-screen takeover with large text, high-contrast colors, loud ringtone
Layer 4 — Caregiver SMS
Text sent to designated contact: 'Dad has not confirmed his 1 PM Paracetamol dose'
Layer 5 — Auto-call
App places a voice call with a recorded or text-to-speech reminder message

5.2 Notification Content Design
	•	Large font size — always 18pt minimum on notification cards
	•	Medicine name, dose, and instruction all visible without opening app
	•	DONE / SNOOZE / SKIP action buttons directly on notification
	•	Context line: 'This is your BEFORE LUNCH dose — eat within 5 minutes of taking this'
	•	Countdown shown for time-sensitive reminders (e.g., 'Iftar in 4 min')

5.3 Caregiver & Family Loop
	•	Caregiver account linked to elderly user — read-only daily schedule view
	•	End-of-day summary report sent to caregiver: doses taken, missed, or skipped
	•	Real-time missed-dose alert within configurable window (e.g., if not confirmed in 30 min)
	•	Caregiver can add or edit reminders remotely
	•	Emergency contact call shortcut on main screen

8. Technical Architecture Overview
8.1 Core Engine Components
	•	Reminder Chain Engine: DAG-based task graph where each node has trigger conditions, confirmation states, and escalation rules
	•	Anchor Time Resolver: Converts fuzzy user input ('around 1 PM', 'after Iftar') into precise scheduled times
	•	Adaptive Scheduler: Learns from user confirmation patterns and suggests time adjustments
	•	Caregiver Sync Layer: Lightweight cloud sync of confirmation states and missed-dose events
	•	Offline-First Storage: All reminder data lives on device; cloud is optional backup
8.2 NLP Intent Parser (Conversational Setup)
The parser extracts: Medicine name, dose, timing keyword (before / after / with), anchor event (meal / Iftar / wakeup / sleep), time offset in minutes, and recurrence (daily / weekly / every 6 hours / course duration). Output is a structured JSON reminder chain that feeds directly into the engine.
8.3 Notification Strategy
	•	Android: WorkManager + AlarmManager for reliable offline scheduling
	•	iOS: UNUserNotificationCenter with background fetch for chain updates
	•	Escalation is handled locally — no push server required for base function
	•	Caregiver alerts use SMS gateway (low-cost, no app install on caregiver's side)

Build a smart reminder app called MemoCare for elderly users (primary), young professionals, and school-age children (secondary). The app's core innovation is Linked Reminder Chains — groups of reminders where each step fires only after the user confirms the prior step is done.
