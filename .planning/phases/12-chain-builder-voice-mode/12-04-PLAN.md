**This plan adds:**

- Regex-based NLP parser (extract medicine names, times, frequencies, chain relationships)
- Parsed chain preview panel (primary/5 bg, dotted teal connectors)
- Individual parsed reminder rows with confidence indicators
- Confirm (create reminders directly) / Edit (switch to Build It pre-filled) buttons

**NLP approach:** Regex pattern matching (not ML/AI inference). Handles common patterns:

- "Take [med] at [time]"
- "Take [med] after [meal]"
- "[med] every [frequency]"
- "[med] then [med] [offset] later"
- Numbers for doses ("2 tablets", "500mg")
  </context>

<steps>
1. Create NLP models (Freezed): ParsedChain (items, overallConfidence, rawText), ParsedReminder (medicineName, dose, unit, time, frequency, anchorType, offsetMinutes, relationship, confidence, warnings).

2. Create NlpParser class: Main parse(String text) method. Split text by commas, "then", "and". For each segment:
   a. Extract medicine name (noun phrase, often first/last word)
   b. Extract time ("at 8am", "at 14:00", "morning", "evening")
   c. Extract meal anchor ("after breakfast", "before lunch", "with dinner")  
   d. Extract dose ("2 tablets", "500mg", "1 capsule")
   e. Extract frequency ("daily", "twice a day", "every morning")
   f. Extract chain keywords ("then", "followed by", "after that")
   g. Calculate confidence based on matches found

3. Write unit tests for NlpParser: Test common phrases, edge cases, multi-medicine chains, ambiguous input.

4. Create ParsedReminderRow: Row widget. Time (left, bold), medicine name + dose (center), confidence check/warning (right). Tap to edit inline.

5. Create dotted teal connector: CustomPainter between rows. Vertical line with 6px teal (#2DD4BF) circles.

6. Create ParsedChainPreview: Card with primary/5 bg. "Here's what I understood:" header. List of ParsedReminderRow with connectors between them.

7. Create ConfirmParseButtons: "Looks right?" text + Row("Edit" outlined button, "Confirm" primary button). Edit → pre-fill Build It form with parsed values + switch tab. Confirm → iterate parsed items → create ReminderData for each → schedule alarms → navigate to /home.

8. Update VoiceModePanel: After transcript done → call NlpParser.parse() → if success: show ParsedChainPreview + ConfirmParseButtons. If failure: show error + "Try again" + "Switch to manual" buttons.

9. Update VoiceModeNotifier: Add parsedChain state, parse() method, confirmParse(), editParse() methods.

10. Widget tests: Verify preview renders parsed items, connectors show, confidence indicators work, confirm creates reminders, edit switches to Build It.
