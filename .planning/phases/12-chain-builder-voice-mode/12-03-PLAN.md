<steps>
1. Add speech_to_text to pubspec.yaml. Run flutter pub get.

2. Create VoiceModeState (Freezed): status enum (idle/listening/processing/done/error), transcript (String), partialTranscript (String), errorMessage (String?).

3. Create VoiceModeNotifier: Initialize SpeechToText. startListening() → request mic permission → start listening with onResult callback → update transcript. stopListening(). retry(). clear(). Handle errors.

4. Create MicButton: 72px FloatingActionButton. Idle: mic icon. Listening: stop icon + AnimatedBuilder pulse rings (3 concentric circles expanding outward with opacity fade). Processing: CircularProgressIndicator.

5. Create WaveformVisualizer: Row of 5 AnimatedContainer bars. Each bar has different animation duration/phase for visual variety. Heights update based on audio level (or simulated random for demo). Only visible when status == listening.

6. Create ExamplePhraseChips: Wrap widget with horizontal scroll. Each chip: ActionChip with text. onPressed → set transcript to chip text → optionally auto-submit. Visible when status == idle.

7. Create TranscriptDisplay: Container with text. Partial results in grey (AppColors.skippedGrey), final results in AppColors.primary. "Analyzing..." text when processing.

8. Build VoiceModePanel: Column → Spacer → MicButton + WaveformVisualizer → ExamplePhraseChips (or TranscriptDisplay if transcript exists) → Spacer. Centered layout.

9. Update AddReminderScreen: "Say It" tab now renders VoiceModePanel instead of placeholder. When transcript captured (status == done), pass to NLP parser or show with "Switch to Build It to edit manually" prompt.

10. Widget tests: Verify mic button state transitions, waveform renders 5 bars, chips render, transcript updates.
    </steps>
