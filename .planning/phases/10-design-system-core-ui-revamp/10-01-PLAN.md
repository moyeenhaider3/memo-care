<objective>
Build the design system foundation that ALL other Phase 10+ plans depend on.

This plan establishes:

1. **Color system** — Navy primary (#1A3C5B) replacing teal, with accent, success, warning, danger palette
2. **Typography** — Inter font family replacing Noto Sans, with design-accurate sizes (H1=28pt, Body=17pt, Caption=14pt)
3. **Component tokens** — Border radii, button heights, touch targets, shadows as reusable constants
4. **Bottom navigation** — 5-tab bar (Home/Schedule/Add/History/Profile) replacing 3-tab bar
5. **FAB** — Centered floating action button for quick-add reminder
6. **Router restructure** — 4 shell branches + /add-reminder + /schedule routes

This is the MOST CRITICAL plan in Phase 10. Every subsequent plan (10-02 through 10-09) depends on these tokens existing. The old color/font values are aliased with @Deprecated so existing screens don't break while they await individual revamps.

Output: Complete design system, bottom nav, FAB, and router — app compiles and runs with new navigation structure. Existing screens render with new theme colors/font.
</objective>

**Current state:**

- Theme: `_buildTheme()` in app.dart uses `ColorScheme.fromSeed(seedColor: Colors.teal)` and `GoogleFonts.notoSansTextTheme()`
- Colors: `AppColors` class in `app_theme.dart` has teal-based WCAG AAA constants
- Bottom nav: 3-tab NavigationBar (Home/History/Settings) in `app_shell.dart`
- Router: 3 StatefulShellRoute branches (/home, /history, /settings)
- No FAB exists
- No /schedule or /add-reminder routes exist

**What this plan changes:**

- Every color, font, and component token across the entire app
- Navigation structure from 3 tabs to 5 tabs
- Router from 3 branches to 4 branches + standalone route
- Adds centered FAB

**Backward compatibility approach:**
Keep old `AppColors.primary`, etc. as `@Deprecated` getters pointing to new values so existing screens don't break. Each subsequent plan (10-02 through 10-08) will update its screen to use new tokens directly and remove deprecated usage.
</context>
