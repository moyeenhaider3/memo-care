<objective>
Comprehensive Semantics and TalkBack focus order pass across every screen in the app (A11Y-04, A11Y-05).

This is the final and most critical accessibility plan. It depends on 08-01 (TTS adds new widgets needing Semantics), 08-02 (UndoBar needs liveRegion Semantics), and 08-03 (layout changes may have moved/added interactive elements).

Two priorities:

1. **Semantics labels (A11Y-04):** Every interactive widget gets a natural-language label. Not "btn_07" but "Confirm taking Metformin 500mg". Labels should make sense when read aloud without context.
2. **Focus order (A11Y-05):** TalkBack navigates using focus order. Without explicit OrdinalSortKey, Flutter uses render order, which may differ from visual layout (e.g., a Row with trailing icon may read the icon before the text). Every screen sets sequential sortKeys top-to-bottom, left-to-right.

The UndoConfirmationBar (from 08-02) sets `liveRegion: true` so TalkBack auto-announces it when it appears — the user doesn't need to navigate to it.

Decorative elements (non-interactive icons, dividers, animations) are explicitly excluded from the Semantics tree to reduce TalkBack noise.

Output: All screens updated + Semantics verification tests.
</objective>

**Focus order pattern (OrdinalSortKey):**

```dart
// Top-to-bottom, left-to-right. AppBar = 0, content = 1-99, BottomNav = 100+
Semantics(
  sortKey: const OrdinalSortKey(1),
  child: NextPendingHeroCard(...),
)

Semantics(
  sortKey: const OrdinalSortKey(2),
  child: Text('Today\'s Schedule'),
)

// For dynamic lists, use offset:
for (int i = 0; i < reminders.length; i++)
  Semantics(
    sortKey: OrdinalSortKey(10.0 + i),
    child: ReminderListTile(reminder: reminders[i]),
  )

// BottomNav always last:
// Items internally labeled with sortKeys 100, 101, 102
```
