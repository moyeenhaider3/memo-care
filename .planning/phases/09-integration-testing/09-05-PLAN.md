  <action>
Create `.planning/phases/09-integration-testing/OEM_TEST_CHECKLIST.md` with detailed step-by-step manual test procedures for each OEM.

The checklist must cover these test categories for EACH OEM (Xiaomi, Samsung, Huawei):

**Category A: Alarm Reliability**

1. Schedule reminder 5 min from now → lock device → wait → verify alarm fires
2. Schedule reminder 30 min from now → lock device → wait → verify alarm fires
3. Schedule reminder → close app from recents → wait → verify alarm fires
4. Schedule reminder → force stop app → wait → document if alarm fires (expected: may not fire after force stop)
5. Schedule 5 reminders within 10 min window → verify all fire

**Category B: Boot Rescheduling**

1. Schedule 3 reminders (5 min, 15 min, 30 min from now)
2. Reboot device
3. After boot, wait for each reminder time
4. Verify all 3 fire at correct times

**Category C: OEM Battery Optimization**
Per-OEM steps:

_Xiaomi (MIUI):_

1. Open app → verify AutoStart guidance screen appears during onboarding
2. Follow AutoStart instructions → verify AutoStart is enabled for MemoCare
3. Verify MemoCare is NOT in "Battery Saver" restricted list
4. Lock device for 15 min → verify alarm fires (battery saver would kill it)
5. Verify "Lock app" in recents is suggested in guidance

_Samsung (OneUI):_

1. Open app → verify battery optimization guidance during onboarding
2. Follow instructions to add MemoCare to "Unmonitored apps" list
3. Verify MemoCare is NOT in "Sleeping Apps" list
4. Leave app unused for 3 days (or simulate via settings) → verify alarms still fire
5. Check "Adaptive Battery" settings → verify MemoCare is excluded

_Huawei (EMUI):_

1. Open app → verify App Launch Manager guidance during onboarding
2. Set MemoCare to "Manage manually" in App Launch Manager (enable auto-launch, secondary launch, run in background)
3. Verify MemoCare is NOT flagged as "power-intensive"
4. Lock device for 15 min → verify alarm fires
5. Check "Battery Optimization" → verify MemoCare is set to "Don't optimize"

**Category D: Escalation Behavior**

1. Schedule reminder → let it escalate to AUDIBLE tier → verify sound plays
2. Continue ignoring → verify FULLSCREEN tier launches (or high-priority notification)
3. Verify wakelock keeps screen on during fullscreen escalation
4. Verify alarm sound loops until user acts

**Category E: Permission Flows**

1. Fresh install → verify each permission is requested appropriately for the device's Android version
2. Deny notification permission → verify degradation messaging
3. Re-grant notification permission → verify recovery
4. On Android 14+ devices: verify full-screen intent permission behavior
5. Verify exact alarm permission matches device Android version

**Category F: Cross-Version Testing**

1. Test on Android 12 (API 31) — verify SCHEDULE_EXACT_ALARM behavior
2. Test on Android 13 (API 33) — verify POST_NOTIFICATIONS runtime permission
3. Test on Android 14 (API 34) — verify USE_FULL_SCREEN_INTENT handling
4. Test on Android 15 (API 35) — verify background activity launch restrictions

Format each test as:

```markdown
### TEST-ID: Description

**Device:** [OEM] [Model] [Android Version] [OEM UI Version]
**Pre-conditions:** [what must be set up before test]
**Steps:**

1. Step one
2. Step two
3. ...
   **Expected Result:** [what should happen]
   **Pass Criteria:** [specific condition that must be true]
```
