<task type="auto">
  <name>Task 2: Configure analysis_options.yaml with strict lint rules</name>
  <files>analysis_options.yaml</files>
  <action>
Replace the entire contents of `analysis_options.yaml` with the following configuration:

```yaml
include: package:very_good_analysis/analysis_options.yaml

analyzer:
  plugins:
    - custom_lint
  exclude:
    - "**/*.g.dart"
    - "**/*.freezed.dart"
  errors:
    invalid_annotation_target: ignore

linter:
  rules:
    public_member_api_docs: false
```

**Key decisions and WHY:**

1. `include: package:very_good_analysis/analysis_options.yaml` — replaces the default flutter_lints. very_good_analysis is stricter and catches more issues.

2. `plugins: [custom_lint]` — required for riverpod_lint to work. riverpod_lint registers itself as a custom_lint plugin via its `pubspec.yaml`. You do NOT add `riverpod_lint` here directly — it auto-registers.

3. `exclude: ["**/*.g.dart", "**/*.freezed.dart"]` — generated files from build_runner (Drift, Freezed, Riverpod, JSON) must be excluded from analysis. They don't follow our lint rules and cannot be modified.

4. `errors: { invalid_annotation_target: ignore }` — Freezed uses annotations on parameters that the analyzer flags as invalid targets. This is a known Freezed issue; the ignore is standard practice.

5. `public_member_api_docs: false` — very_good_analysis requires dartdoc on all public members. This is too strict for an app (it's designed for packages). Disable it.

**DO NOT** add any other rule overrides unless `flutter analyze` explicitly fails on a rule that cannot be fixed in the source code.
</action>
