**type_converters.dart** — Create a `DateTimeConverter` that converts between `DateTime` and `int` (UTC epoch milliseconds):

```dart
import 'package:drift/drift.dart';

/// Converts DateTime to/from UTC epoch milliseconds (int).
/// All times in the database are stored as UTC epoch millis (P-10).
class DateTimeConverter extends TypeConverter<DateTime, int> {
  const DateTimeConverter();

  @override
  DateTime fromSql(int fromDb) =>
      DateTime.fromMillisecondsSinceEpoch(fromDb, isUtc: true);

  @override
  int toSql(DateTime value) => value.toUtc().millisecondsSinceEpoch;
}
```

**AppDatabase class:**

```dart
@DriftDatabase(tables: [
  ReminderChains,
  Reminders,
  ChainEdges,
  Confirmations,
  MealAnchors,
])
class AppDatabase extends _$AppDatabase {
  AppDatabase() : super(_openConnection());

  // IMPORTANT: Start at version 1. Increment this number for each migration.
  // Migration framework from day 1 (P-03).
  @override
  int get schemaVersion => 1;

  @override
  MigrationStrategy get migration => MigrationStrategy(
        onCreate: (m) async {
          await m.createAll();
        },
        onUpgrade: (m, from, to) async {
          // Future migrations go here.
          // Example: if (from < 2) { await m.addColumn(reminders, reminders.newColumn); }
        },
      );
}

LazyDatabase _openConnection() {
  return LazyDatabase(() async {
    final dbFolder = await getApplicationDocumentsDirectory();
    final file = File(p.join(dbFolder.path, 'memo_care.sqlite'));
    return NativeDatabase.createInBackground(file);
  });
}
```

Required imports at the top:

```dart
import 'dart:io';
import 'package:drift/drift.dart';
import 'package:drift/native.dart';
import 'package:path/path.dart' as p;
import 'package:path_provider/path_provider.dart';
import 'type_converters.dart';

part 'app_database.g.dart';
```

**IMPORTANT:** `path` is a transitive dependency of path_provider — do NOT add it to pubspec.yaml separately. It's already available.

**WHY DateTimeConverter on time columns:** P-10 mandates UTC epoch millis for all times. Using Drift's built-in DateTime would store as Unix seconds or ISO string — inconsistent.

**WHY NativeDatabase.createInBackground:** Opens the database on a background isolate, preventing jank on the main thread during large queries.

**WHY MigrationStrategy from day 1:** P-03 warns that adding migration support after users have data is fragile. The framework must exist even if onUpgrade is empty now.

**database_provider.dart** — Create the Riverpod provider for the database singleton:

```dart
import 'package:riverpod_annotation/riverpod_annotation.dart';
import '../database/app_database.dart';

part 'database_provider.g.dart';

/// Provides the singleton [AppDatabase] instance.
/// keepAlive: true ensures the database outlives individual screens.
@Riverpod(keepAlive: true)
AppDatabase appDatabase(Ref ref) {
  final db = AppDatabase();
  ref.onDispose(db.close);
  return db;
}
```

Note: The `Ref` type is unqualified — Riverpod 3.x uses the base `Ref` type in generated providers. Do NOT use `AppDatabaseRef` — that's the old API.

**WHY WidgetsFlutterBinding.ensureInitialized():** Required before accessing path_provider (used by Drift's LazyDatabase) and before any plugin initialization.

**WHY ProviderScope at root:** All Riverpod providers need a ProviderScope ancestor. Placing it at the root ensures the database provider (keepAlive: true) lives for the entire app lifecycle.

**WHY minimal App:** Phase 01 is foundation only. GoRouter, screens, and theming come in later phases. The app just needs to launch, initialize the database, and prove the stack works.

**DO NOT** eagerly initialize the database in main(). The LazyDatabase opens on first access via the Riverpod provider. This is intentional — it keeps startup fast and defers the IO to when it's actually needed.
