Reference: `kids-mode-dashboard.html` (Stitch) + P12 (docx)

This plan creates an entirely new feature module: `lib/features/kids_mode/`.
Can run in parallel with 11-03 (Ramadan) since they're independent modules.

Output: Complete kids dashboard with theme, navigation, checklist, mascot, and points.
</objective>

**Nothing exists yet.** This is a completely new feature module.

**10-01 provides:** Base design tokens (AppColors, AppTypography, etc.) that KidsTheme overrides.

**Key build:**

- Kids theme (purple palette independent from main navy theme)
- Custom 3-tab bottom nav (Quest/Prizes/Me)
- Dashboard with quest progress, animated task cards, mascot
- Points system with earning/display
- Star rating based on daily completion
- Parent view toggle with PIN protection
  </context>

<steps>
1. Create KidsTheme: ThemeData with purple primary, yellow secondary, larger Inter text. Export as KidsTheme.data.

2. Create kids_models.dart: Quest (id, title, icon, isCompleted, points), KidsPoints (total, today, streak). Freezed models.

3. Create KidsModeState (Freezed): quests (List<Quest>), points (KidsPoints), dailyRating (int 0-5), isParentView (bool).

4. Create KidsModeNotifier: Loads today's reminders as quests, tracks completions, calculates points, manages parent toggle PIN.

5. Create KidsBottomNav: 3-tab NavigationBar (Quest/Prizes/Me). Purple active color. Rounded top border.

6. Create QuestProgressBar: AnimatedContainer horizontal bar. Shows "X of Y quests". Gradient fill.

7. Create TaskCard: Card with icon, name, time, animated checkbox. Green check + strikethrough when done. Purple subtle glow when pending.

8. Create DailyChecklist: Column of TaskCard widgets from today's quests.

9. Create MascotWidget: Placeholder icon/SVG with bobbing animation (AnimatedBuilder + Transform.translate). Speech bubble with random encouraging text.

10. Create PointsDisplay: Star icon + count. Animated increment on point earn.

11. Create StarRating: 5 stars row. Filled based on % completion. Animated fill.

12. Create ParentViewToggle: IconButton → PIN dialog (4-digit) → switches isParentView state → redirects to /home.

13. Build KidsDashboardScreen: Theme(data: KidsTheme.data) → Scaffold with KidsBottomNav → AppBar (greeting + PointsDisplay) → QuestProgressBar → DailyChecklist → MascotWidget.

14. Update AppRouter: Add /kids shell route with 3 branches. Redirect /home → /kids/quest when kids mode active.

15. Widget tests: Verify theme applies, tasks render, points increment, parent toggle shows PIN dialog.
    </steps>
