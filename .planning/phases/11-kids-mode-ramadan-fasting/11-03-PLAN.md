<steps>
1. Create RamadanTheme: ThemeData with dark background #0D1B2A, card #1A2E44, primary gold #F0A500, white text. Export as RamadanTheme.data.

2. Create fasting_models.dart (Freezed): FastingDay (dayNumber, sehriTime, iftarTime, isFasting), FastingMedicine (medicine data + sehri/iftar grouping).

3. Create FastingState (Freezed): currentDay, sehriTime, iftarTime, isFasting, sehriMedicines, iftarMedicines, progressPercent.

4. Create FastingNotifier: Loads fasting config from settings. Groups medicines into sehri/iftar based on anchor times. Calculates progress (current time position between sehri and iftar).

5. Create StarDotPattern: CustomPainter that draws small star/circle shapes at random positions with low opacity on dark background.

6. Create RamadanHeader: Row with crescent moon icon (gold) + Column ("Ramadan Mubarak" 22pt gold, "Day X of 30" 14pt white).

7. Create SehriIftarCards: Two Cards in a Row. Each card: colored top border (4px), time display, countdown Timer (hours:minutes). Dark card bg. Blue for sehri, gold for iftar.

8. Create FastingProgressBar: Container with gradient LinearGradient (sehri-blue → iftar-gold). Position dot: Positioned widget at calculated offset. Dot has BoxShadow glow. Percentage label below.

9. Create MedicineSection: Section header + list of medicine cards. Each medicine: name, dose, time, DONE/SKIP. Dark card styling. Gradient vertical timeline line connecting items.

10. Build RamadanScreen: Theme(data: RamadanTheme.data) → Scaffold → Stack (StarDotPattern bg) → SafeArea Column (RamadanHeader → SehriIftarCards → FastingProgressBar → MedicineSection(sehri) → MedicineSection(iftar)).

11. Update AppRouter: Add /ramadan route.

12. Widget tests: Verify dark theme applies, countdown ticks, progress bar renders, medicines grouped correctly.
    </steps>
